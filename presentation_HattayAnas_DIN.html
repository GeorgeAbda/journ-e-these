How Do We Model a Workflow?

Definition

A workflow is a sequence of connected tasks that must be executed in a specific order to complete a job.

Tasks: Individual units of work (e.g., process data, train model)
Dependencies: Task A must finish before Task B can start
DAG: Directed Acyclic Graph representation

Example: Data Processing Pipeline

1
Read Data
2
Clean Data
3
Analyze

Each task depends on the previous one completing successfully

What is Cloud Scheduling?

The Problem

  • Multiple tasks need computing resources
  • Limited number of machines (VMs)
  • Dependencies create constraints
  • Energy consumption matters

The Decision

For each ready task:
🤔 Which VM should run it?
⏰ When should it start?
How much energy will it use?

The Goals

  • Fast: Minimize completion time
  • Efficient: Minimize energy use
  • Cheap: Minimize cost
  • Correct: Respect dependencies

The Challenge: It's a Tradeoff!

Faster machines often consume more energy. Parallel execution can reduce time but increase total power usage. There's no perfect solution—only smart compromises.

What are Graph Neural Networks?

Traditional AI

• Works with flat data (tables, images)
• Ignores relationships between items
• Fixed input size
• Example: "Task A needs 4 cores"

Graph Neural Networks

• Works with connected data (graphs)
• Learns from relationships
• Flexible input size
• Example: "Task A depends on Task B"

Why GNNs for Scheduling?

Workflows are naturally graphs! GNNs can understand that "Task A must finish before Task B" and make smarter scheduling decisions based on the entire workflow structure, not just individual tasks.

Markov Decision Process (MDP) Formulation

State (s)

  • Ready tasks, DAG dependencies
  • VM statuses (CPU, Mem, Energy)
  • Queue lengths, time, DVFS state

Action (a)

  • Assign task i → VM j
  • Optionally set DVFS level
  • Defer when no feasible mapping

Reward (r)

  • − Energy per step (Wh)
  • − Makespan penalty
  • Constraints: deadline violations

Transition (P)

P(s'|s,a) evolves by simulator; stochastic job times.

Policy (π)

π(a|s) selects VM and DVFS to optimize energy–makespan.

Objective

Maximize E[∑ γ^t r_t] subject to SLA/deadline constraints.

Other Solutions

Graph Neural Networks

how:

  • • Model the entire scheduling problem as a bipartite graph, where nodes represent machines and tasks, and edges represent possible assignments
  • • By passing messages between nodes, GNNs can learn how the state of one candidate affects others

Convolutional Neural Networks

how:

  • • Here, we treat the whole set of machine-task candidates as if it were an image where each "pixel" holds the features of a candidate
  • • To make this work, we need to fix the maximum number of candidates and use masking techniques for jobs with fewer candidates.

Visualization Approach

During training, and to get a clearer sense of how an agent actually makes its scheduling decisions, we visualized its activation boundaries in a two-dimensional PCA-reduced feature space for a representative state.

Overfitting Detection

Good Boundaries

Smooth, generalizable regions

Overfitting

Jagged, fragmented regions

When a neural network is overfitting, its decision boundaries often become jagged and overly detailed.

Practical Application

Whenever we noticed boundaries that were overly fragmented or too complex, it signaled possible overfitting or unstable decision logic. To address this, we made adjustments to the neural network architecture to encourage the formation of smoother and more generalizable decision regions.
